# GitHub Pull Request 제목 및 설명 생성

현재 브랜치에 대해 GitHub Pull Request에 작성할 제목과 description을 제안해주세요.

## 분석 단계

1. `git branch --show-current`로 현재 브랜치 확인
2. `git log main..HEAD --oneline`으로 main 브랜치 이후 커밋 확인
3. `git diff main...HEAD --stat`로 변경된 파일 통계 확인
4. `git diff main...HEAD`로 상세 변경사항 확인 (필요시)
5. `git log -10 --oneline`으로 최근 커밋 스타일 참고

## PR 제목 형식

Conventional Commits 양식을 따르되, 스코프는 사용하지 않고 한글로 작성:

<type>: <description>

### Type 종류

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링, 성능 개선
- `style`: 코드 스타일 변경
- `test`: 테스트 추가 또는 수정
- `docs`: 문서 변경
- `chore`: 빌드, 설정, 의존성 등 기타 변경

### 제목 작성 규칙

- 50자 이내로 간결하게
- 명령형 또는 명사형
- 마침표 없음

### 제목 예시

feat: 북마크 추가 기능 구현  
fix: 그리드 스크롤 결함 해결  
refactor: ERP 트리 상태 관리 개선  

## PR Description 형식

반드시 아래 마크다운 형식을 따라야 합니다:

**왜 변경하려 하시나요?**

(변경 이유를 명확하게 작성)

**어떻게 변경하려고 하나요?(optional)**

- N/A

**해당 변경으로 영향받거나 검토해야 할 사항은 무엇인가요?(optional)**

- N/A

**머지하기 전에 반드시 확인되어야할 사항이 있다면 작성해 주세요.(optional)**

- [ ] N/A

### Description 작성 가이드

1. 왜 변경하려 하시나요? (필수)
   - 이 PR의 목적과 배경 설명
   - 해결하려는 문제나 추가하려는 기능

2. 어떻게 변경하려고 하나요? (선택)
   - 주요 변경사항을 간략히 나열
   - 기술적인 접근 방법

3. 해당 변경으로 영향받거나 검토해야 할 사항은 무엇인가요? (선택)
   - 다른 기능에 미치는 영향
   - 특별히 검토가 필요한 부분

4. 머지하기 전에 반드시 확인되어야할 사항이 있다면 작성해 주세요. (선택)
   - 체크리스트 형식 (- [ ])
   - 테스트, 빌드, 배포 전 확인사항 등

## 출력 형식

분석 후 다음과 같이 출력해주세요:

1. 변경사항 요약 (브랜치명, 커밋 수, 변경된 파일 수)
2. 제안하는 PR 제목 (2-3개 옵션)
3. 추천 제목 (가장 적합한 것 1개 선택)
4. PR Description (위의 마크다운 형식으로)

## 출력 예시

변경사항 요약
- 브랜치: feature/bookmark-add
- 커밋 수: 3개
- 변경된 파일: 8개

제안하는 PR 제목

옵션 1 (추천):
feat: 북마크 추가 기능 구현

옵션 2:
feat: ERP 메뉴 북마크 기능 추가

PR Description

**왜 변경하려 하시나요?**

사용자가 자주 사용하는 ERP 메뉴를 빠르게 접근할 수 있도록 북마크 기능을 추가합니다.

**어떻게 변경하려고 하나요?(optional)**

- 사이드바에 북마크 섹션 추가
- 드래그 앤 드롭으로 북마크 순서 변경 기능
- 로컬 스토리지를 활용한 북마크 상태 저장

**해당 변경으로 영향받거나 검토해야 할 사항은 무엇인가요?(optional)**

- 사이드바 레이아웃 변경으로 기존 메뉴 위치 조정
- 로컬 스토리지 용량 확인 필요

**머지하기 전에 반드시 확인되어야할 사항이 있다면 작성해 주세요.(optional)**

- [ ] 북마크 추가/삭제 동작 확인
- [ ] 드래그 앤 드롭 정렬 동작 확인
- [ ] 다양한 브라우저에서 테스트
