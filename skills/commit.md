# 커밋 메시지 생성

현재 git 변경사항을 분석하여 Conventional Commits 형식의 커밋 메시지를 생성해주세요.

## 분석 단계

1. `git status`로 변경된 파일 확인
2. `git diff --cached`로 스테이징된 변경사항 확인
3. 스테이징된 변경사항이 없으면 `git diff`로 모든 변경사항 확인
4. `git log -5 --oneline`으로 최근 커밋 스타일 참고

## 커밋 메시지 형식

이 프로젝트는 Conventional Commits 형식을 따릅니다:

<type>: <description>

[optional body]

### Type 종류

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링, 성능 개선 (기능 변경 없음)
- `style`: 코드 스타일 변경 (포맷팅, 세미콜론 등)
- `test`: 테스트 추가 또는 수정
- `docs`: 문서 변경
- `chore`: 빌드, 설정, 의존성 등 기타 변경

### 작성 규칙

1. Description (설명)
   - 한글로 작성
   - 50자 이내로 간결하게
   - 명사형 또는 서술형으로 간결하게 작성
   - 마침표 없음

2. Body (본문) - 선택사항
   - 변경 이유나 상세 설명이 필요할 때만 추가
   - 한글로 작성

3. 예시

   feat: 북마크 추가 기능 구현

   - ERP 메뉴 북마크 기능 추가
   - 북마크 드래그 앤 드롭 정렬 지원
   - 로컬 스토리지에 북마크 상태 저장

## 출력 형식

변경사항을 분석한 후, 다음과 같이 출력해주세요:

1. 변경사항 요약 (간단히)
2. 제안하는 커밋 메시지 (2-3개 옵션)
3. 추천 메시지 (가장 적합한 것 1개 선택)

사용자가 선택할 수 있도록 옵션을 제공하되, 가장 적합한 메시지를 명확히 추천해주세요.
